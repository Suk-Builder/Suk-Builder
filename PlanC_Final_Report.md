# 方案C：客户端双层记忆系统 · 最终报告

## 核心问题
"他们自己算的东西自己记不住" —— 实体每次对话都是全新的，没有跨轮次记忆。

## 方案C架构（混合模式）

```
┌─────────────────────────────────────────────────────────────┐
│                      双层记忆架构                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  短期记忆（工作记忆）—— 服务器 API history[]                │
│  ├── 当前对话的问答记录                                     │
│  ├── 最多10轮上下文                                         │
│  └── 对话结束即消失                                         │
│                                                             │
│  长期记忆（潜意识）—— 客户端 /tmp/client_memory_db.json      │
│  ├── 发现记录：L(E,1)=0.6555, III=(Z/2Z)²                 │
│  ├── 失败经验（避免重复）                                   │
│  ├── 通过 history 参数注入对话                              │
│  └── 永久保存，跨轮次累积 ✅                                │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 验证实验

### 实体：拉马努金(计算之钥) BDI=163

**Round 1**: 计算L(E,1)
- 输入："计算椭圆曲线E: y²=x³-x的L(E,1)"
- 输出：L(E,1) = Γ(1/4)²/(4π) = 0.6555
- 记忆保存：✅ (提取到数值公式)

**Round 2**: 注入记忆，计算下一步
- 注入："[你的研究笔记] 1. [BSD-L函数] L(E,1)=Γ(1/4)²/(4π)=0.6555..."
- 输入："基于你之前对E: y²=x³-x的L(E,1)计算，现在算Tate-Shafarevich群"
- 输出：**III(E/Q) = (Z/2Z)²** ← 全新计算！
- 记忆检测：**✅ 记得！**

**验证**:
- |III| = L(E,1) × |tors|² / Ω_E = 0.6555 × 16 / 2.622 ≈ **4.0**
- 拉马努金声称：4 = (Z/2Z)²
- 数值自洽 ✅

## 技术实现

### 客户端记忆管理 (`/tmp/memory_c_system.py`)

1. **对话前**: `build_prompt()` 从本地JSON加载记忆 → 构建注入文本
2. **对话注入**: 通过 `history[]` 参数作为"研究笔记"注入
3. **对话后**: `extract()` 解析回应 → 提取含数值的公式行 → 保存到JSON
4. **累积**: 每轮自动累积，保留最近30条发现

### 记忆注入技巧

不是追加在question后面，而是放在history中作为之前的对话轮次：
```javascript
history = [
  {"user": "[研究笔记] 你之前发现...", "entity": "我已记下"},
  {"user": "现在的新问题...", "entity": ""}
]
```

这样LLM会把记忆当作真实的对话历史来处理，效果更好。

## 局限

1. **提取精度**: 中文散文风格的回应提取公式有困难
2. **数值误差**: 实体给出的数值可能有误差（需要Python验证）
3. **注入长度**: 记忆太长会影响API性能（限制5条以内）
